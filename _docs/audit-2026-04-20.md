## Part 1 — Codebase code smells

### HIGH severity

#### 1. `website/settings.py:24` — `ALLOWED_HOSTS = ['*']`
Permissive wildcard allows any `Host` header → opens HTTP Host header attacks and cache poisoning.
Fix: read from env, default to `localhost,127.0.0.1`. Whitelist `aishippinglabs.com`, `dev.aishippinglabs.com`.

#### 2. `website/settings.py:20` — Hardcoded `django-insecure-` SECRET_KEY fallback
Critical if env var is missing in prod. The placeholder from `django startproject` is still in tree.
Fix: when `DEBUG=False`, require `SECRET_KEY` from env without fallback (assert or raise `ImproperlyConfigured`).

#### 3. `payments/views/webhooks.py:58-61` — Fragile event-data extraction
Tries dict access, then `getattr` fallback for Stripe event parsing. If the SDK shape changes, this silently succeeds with garbage.
Fix: validate event structure upfront and raise on parse failure.

#### 4. `integrations/services/github.py:404` — Bare `except` in `_parse_yaml_text`
Catches all exceptions from `yaml.safe_load` and returns `{}` silently. Malformed YAML from the content repo silently becomes empty content.
Fix: catch `yaml.YAMLError` only, log the path + error, and either re-raise or surface in sync result so the failure shows in `/studio/sync/`.

#### 5. `content/views/courses.py:288` — N+1 in `api_course_detail`
Loop does `for unit in module.units.all().order_by('sort_order')` even though `course.get_syllabus()` already calls `prefetch_related('units')` on line 280. The extra `.all().order_by()` busts the prefetch cache.
Fix: iterate `module.units.all()` only, and push ordering into the prefetch (`Prefetch('units', queryset=Unit.objects.order_by('sort_order'))`).

#### 6. `content/views/courses.py:357` — Same N+1 pattern (course detail HTML view)
Mirrors #5. Same fix.

### MEDIUM severity

#### 7. `integrations/services/github.py:225-240, 281-286` — Inconsistent timeouts on GitHub API calls
10s for token fetch, 15s for auth. Not configurable.
Fix: pull both from settings (`GITHUB_API_TIMEOUT`), document defaults in `website/settings.py`.

#### 8. `content/tier_config.py` — Hardcoded tier definitions
Tier metadata likely belongs in DB or settings, not source.
Fix: move to `Tier` model fields or env-driven settings; double-check if this still has callers after the recent tier work.

#### 9. `integrations/middleware.py:32` — Module-level mutable cache `_ANNOUNCEMENT_BANNER_CACHE`
Each gunicorn/uvicorn worker has its own copy. `clear_announcement_banner_cache()` only invalidates one worker — banner edits appear to "not save" on multi-worker dev/prod.
Fix: use Django's cache framework (we already added `DatabaseCache` for `django_q_cache`; same backend works here).

#### 10. `integrations/services/github.py:548` — Bare `except` in `_parse_frontmatter_text`
Same shape as #4. Malformed frontmatter silently becomes `{}`.
Fix: catch specific exceptions, log, surface in sync result.

#### 11. `payments/views/webhooks.py:76-84` — Over-defensive object→dict conversion
Tries `.to_dict()`, isinstance check, `dict()` constructor. Suggests uncertainty about Stripe SDK shape.
Fix: pin the Stripe SDK version, assert type, follow documented patterns.

#### 12. `integrations/services/ses.py:137` — Bare `except` in SES signature verification
Catches all exceptions from cert download/parse — legitimate network errors are indistinguishable from protocol violations / attacker payloads.
Fix: separate `requests.RequestException`, certificate parse errors, signature mismatches; log each category.

#### 13. `content/access.py` — Not yet audited
Access control entrypoint (`can_access`). Worth a manual review for tier-comparison bypasses, especially after the recent CourseAccess work.

#### 14. `content/models/course.py:169-185` — `total_units()` / `completed_units()` issue independent queries
Fine in isolation, but called in loops over Course lists → N+1.
Fix: annotate at queryset level (`Count('modules__units')`) or `prefetch_related('modules__units')` at the call site.

#### 15. `jobs/tasks/youtube_upload.py:166` — Unguarded `s3_client.download_file()`
No partial-download cleanup. Large files or dropped connections leave incomplete temp files but the code proceeds.
Fix: wrap in try/except that deletes the temp file on failure; consider checksum verification.

#### 16. `events/models/event.py` — URLFields with `blank=True, default=''`
Two representations of "no value" (NULL vs empty string). Complicates queries (`isnull=True` vs `=''`).
Fix: pick one (prefer `null=True`, normalize to `None`), data migration to clean existing rows.

#### 17. `payments/views/webhooks.py:85-91` — Webhook records as processed before handler succeeds
If `handler()` raises (DB lock, timeout, downstream API down), it logs but still calls `record_processed_event()`. Stripe sees 200 OK → no retry → silent data loss.
Fix: only record on confirmed success. For non-retryable errors (signature invalid, malformed event), record explicitly with a status flag so we don't infinite-retry.

### LOW severity

#### 18. `website/context_processors.py:14` — `__import__('datetime')` inside function
Slower than module-level import and unusual. Move to top.

#### 19. `website/settings.py:252-257` — GitHub key resolution chain (file → env → Secrets Manager)
Defensive but hard to follow. Add docstring documenting precedence.

#### 20. `integrations/services/github.py:280` — `max_pages = 20` cap with no warning
Silently truncates if a repo has 2000+ installations. Add `logger.warning` when hit.

#### 21. `content/views/home.py:445-451` — `_sort_key()` recreates datetimes with fallback
Defensive dating logic suggests inconsistent date vs datetime in models.
Fix: normalize at model layer (always `DateTimeField`), drop the fallback.

#### 22. `payments/views/checkout.py` — Repeated `json.loads(request.body)` try/except (lines 45, 94, 134…)
Extract to helper `_get_json_body(request)`.

#### 23. `content/models/course.py:9-26` — `render_markdown()` duplicated in `events/models/event.py`
Move to `content/utils/markdown.py` (or similar shared module).

#### 24. `integrations/models/content_source.py` — `unique_together` missing `content_path`
Multiple sources pointing at same `(repo, type, path)` would collide silently during sync.
Fix: extend `unique_together` to include `content_path`.

#### 25. `studio/views` — Not fully audited
Specifically the worker-health and dashboard views. Audit for query perf, cache invalidation races, admin-only access.

## Part 2 — Test quality

### TL;DR
Suite is well-structured overall. Webhook tests are exemplary (idempotent, comprehensive, correct mocking). Access control is well-layered (Django + Playwright). Main weaknesses: substring `assertIn` against full HTML, status-code-only assertions, hard-coded Playwright waits, scattered fixture setup, and a few critical-path tests missing `@tag('core')`.

### Guideline violations

#### T1. `accounts/tests/test_auth.py:19-62` — Substring `assertIn` on full HTML body
e.g. `self.assertIn("Sign in with Google", content)` matches anywhere in the page (header, footer, JS, even an analytics event name). If Google button is removed but "Sign in with Google" appears elsewhere, test still passes.
Fix: `assertContains(response, '<button data-provider="google">Sign in with Google</button>', html=True)` or assert on `response.context['form']`.

#### T2. `studio/tests/test_access.py:32-169` — 18 status-code-only assertions
No assertion on redirect target, body, or context. A 302 to the wrong place still passes; a 200 page that leaks data still passes.
Fix: add `self.assertIn('/accounts/login/', response.url)` for 302s, content checks for 200s.

#### T3. `accounts/tests/test_models.py:11-24, 67-71` — Tests Django framework behavior
`test_tier_defaults_to_free` tests a `ForeignKey(default=...)`. `test_create_user` round-trips Django's `AbstractBaseUser`. Per guidelines, don't test ORM/framework behavior.
Fix: delete; keep only tests for custom `save()`, `clean()`, computed properties, custom managers.

#### T4. `voting/tests/test_models.py:104-143` — `setUp` for read-only fixtures
`PollOptionModelTest` and `PollVoteModelTest` re-create poll/option/user per test method.
Fix: move to `@classmethod setUpTestData(cls)`.

#### T5. Fixture duplication across studio tests
Same staff user setup appears in `studio/tests/test_access.py:19-26`, `test_courses.py:23-28`, `test_downloads.py`, `test_events.py`, etc.
Fix: `StaffUserMixin` in `tests/fixtures.py`.

### False-positive risk

#### T6. `payments/tests/test_checkout.py:35-43` — `test_requires_authentication` checks only status code
Doesn't verify no `StripeCheckoutSession` was created. If endpoint silently creates a session before checking auth, test passes.
Fix: snapshot count before, assert unchanged after.

#### T7. `accounts/tests/test_auth.py:115-134` — Header tests don't assert status code first
A 500 error containing "Sign in" in a stack trace would pass.
Fix: assert `status_code == 200` first, then content.

### Brittleness

#### T8. `playwright_tests/test_github_content_sync.py:1322, 1336` — Hard-coded `wait_for_timeout(1000/1100)`
Will randomly fail on slow CI.
Fix: `page.locator(".sync-complete-badge").wait_for(state="visible", timeout=5000)`.

#### T9. Time-sensitive tests without `freezegun`
e.g. `content/tests/test_recordings.py:33` uses `timezone.now() - timedelta(days=7)`. At UTC midnight boundary or under load, edge cases drift.
Fix: `@freeze_time("2026-04-20 12:00:00")`.

### Coverage gaps

#### T10. Webhook idempotency missing email/log assertions
`payments/tests/test_webhooks.py:623-647` checks tier doesn't change on duplicate event but doesn't check `WebhookEvent.objects.count() == 1` or `len(mail.outbox) == 1`.

#### T11. `payments/tests/test_webhooks.py:149-166` — Missing negative assertions
On fresh checkout, asserts tier updated but not that `billing_period_end` is `None` (since not a renewal).

#### T12. Soft-delete sync coverage gap
No test asserts that when a GitHub webhook deletes a content file, the corresponding row gets `deleted_at` set and disappears from catalog. Critical path.

#### T13. Webhook signature validation — missing tampered-payload test
`payments/tests/test_webhooks.py` has signature tests, but no explicit test for "valid sig, then payload modified" rejection.

#### T14. No concurrent-webhook tests
Two identical events in flight simultaneously — does the unique constraint on `WebhookEvent.stripe_event_id` actually prevent dupes under race?

### Layering

#### T15. `playwright_tests/test_foldable_sidebar.py:29-40` — Manual `connection.close()`
Workaround for pytest-django transaction isolation. Use `@pytest.mark.django_db` properly instead.

### Other

#### T16. `content/tests/test_access_control.py:143-163` — `test_tier_access_matrix` tests 13 things in one
Uses `subTest`, which is acceptable, but borderline. If 12th fails, it's easy to miss in CI summary.
Fix: parameterize or split.

#### T17. `payments/tests/test_webhooks.py:263-281` — Doesn't assert log entry on bad tier
`test_no_error_when_tier_not_found` checks no crash but not that we logged the warning.
Fix: `with self.assertLogs('payments.services', level='WARNING') as log:`.

#### T18. Missing `@tag('core')` on critical tests
`accounts/tests/test_tier_override.py` not tagged at class level. `make test-core` skips genuinely critical tests.
Fix: add `@tag('core')` to the base class.

#### T19. `accounts/tests/test_auth.py:64-69` — `test_login_page_redirects_authenticated_user`
Checks `response.url == "/"` but doesn't `follow=True` to verify redirect target is actually accessible and user is still logged in.

#### T20. `tests/fixtures.py` — `TierSetupMixin` not consistently used
Some tests `Tier.objects.get(slug='basic')` (assumes migration seeded), others `Tier.objects.create()`. Inconsistent.
