a:T486,<p><img src="/images/projects/to-do-list-agent.png" alt="my-daily-tasks-agent repository on GitHub"></p>
<p>Reference project from the first cohort of the <a href="https://maven.com/alexey-grigorev/from-rag-to-agents">AI Engineering Bootcamp</a>, built to help students understand the requirements for a robust, end-to-end AI application.</p>
<p>The agent does not use RAG or a knowledge base. Instead, it relies on the backend's OpenAPI specification: the spec is fed to the model so it can create tools to get tasks or mark them as complete. Monitoring is done with <strong>Logfire</strong>, which tracks the entire session, including the specific tools used and the cost of each interaction. The project includes <strong>18 pytest tests</strong> covering different scenarios, such as checking whether the right tool is invoked when asking about today's tasks.</p>
<p><strong>Tech stack:</strong> Lovable (frontend), FastAPI (Python backend), OpenAPI, Logfire, pytest.</p>
