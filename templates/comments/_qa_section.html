{% comment %} Shared comments thread partial. Used by the course unit Q&A surface, the workshop tutorial Q&A surface, and the personal plan comments surface (issue #499). The plan surface customizes the section heading and composer copy via template parameters; the JavaScript IIFE in ``comments/_qa_script.html`` reads ``data-content-id`` on the wrapper and is identical across surfaces. Parameters: - ``content_id`` (required): UUID matching the content's ``content_id`` / ``Plan.comment_content_id`` field. - ``user_authenticated`` (required): boolean truth value of ``request.user.is_authenticated``. - ``section_heading`` (optional, default "Questions & Answers"): the heading text rendered above the thread. - ``composer_placeholder`` (optional, default "Ask a question about this lesson..."): placeholder text for the new-comment textarea. - ``composer_button_label`` (optional, default "Post Question"): label for the submit button. - ``composer_signin_prompt`` (optional, default "to ask questions"): trailing copy after the "Sign in" anchor in the anonymous branch. - ``composer_disabled`` (optional, default False): when True the authenticated branch hides the textarea and renders a "discussion not available" notice instead. Used by private-plan owner pages where comments are visible but only staff can write. - ``composer_disabled_reason`` (optional): the text shown in the disabled notice. Defaults to a generic message. {% endcomment %}
