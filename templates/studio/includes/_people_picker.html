{% comment %} Reusable Studio people picker (issue #720). Renders a search input backed by the staff JSON endpoint ``studio_user_search`` (``/studio/api/users/search/``). Suggestion rows show the user's display name (bold), email (muted), and contextual badges: ``In this sprint`` and ``Has plan in sprint`` when a sprint is in context, ``No community access`` when the user is below LEVEL_MAIN, and ``Tier: {level_name}`` for the current tier. Block parameters (pass via ``{% include ... with key=value %}``): - ``name`` (required): form field name for the selected user id (hidden input). - ``id_prefix`` (required): DOM id prefix to avoid collisions when several pickers appear on one page (e.g. ``tier-override-user``). - ``search_url`` (required): URL that returns the JSON ``{results: [...]}`` payload. Usually ``{% url 'studio_user_search' %}``. - ``placeholder`` (optional, default ``Search by name or email...``). - ``label`` (optional, default ``User``): visible label above the input. - ``extra_query`` (optional): URL-encoded extra query string appended to every search request (e.g. ``sprint=may-2026``). - ``on_select_action`` (optional, default ``set_value``): one of ``navigate`` (jump to ``navigate_url_template`` with ``{id}`` replaced), ``submit_form`` (set the hidden id then submit the enclosing form), or ``set_value`` (set the hidden id and dispatch a ``picker:select`` event). - ``navigate_url_template`` (required when ``on_select_action='navigate'``): url with a ``{id}`` placeholder. - ``testid_prefix`` (optional, default = ``id_prefix``): override the ``data-testid`` prefix used for input, suggestion list, and rows. Keyboard support: ArrowDown / ArrowUp moves the highlighted suggestion, Enter activates the highlighted row, Esc closes the suggestion list. {% endcomment %} {% with effective_testid_prefix=testid_prefix|default:id_prefix effective_placeholder=placeholder|default:"Search by name or email..." effective_label=label|default:"User" effective_on_select=on_select_action|default:"set_value" %}
